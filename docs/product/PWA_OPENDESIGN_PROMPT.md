# OpenDesign Prompt：Pokemon Champions Tool 环境优先重构

请为 Pokemon Champions Tool 设计一版移动端 PWA 高保真原型。目标不是营销页，而是实际可用的工具型产品界面。请优先做移动端视图，宽度参考 390-430px。

## 产品定位

Pokemon Champions Tool 面向已经有一定宝可梦对战理解的用户。用户可能只是打开看一眼当前环境，也可能保存和编辑自己的队伍，再用伤害计算、速度线和图鉴验证思路。

这个产品不是强引导式组队推荐工具，不要把用户强行带入“AI 推荐队伍”或“必须补强”的流程。文案要中性、克制，使用“参考、样本占比、常同队出现、导入配置、保存、验证、复盘”等表达。避免“最佳队友、推荐搭档、你应该、必带、官方使用率、全环境使用率、最强推荐”等文案。

## 现有 PWA 视觉参考

请继承现有 PWA 的移动端工具气质：深色背景、高对比卡片、紧凑信息密度、底部固定导航、属性胶囊、Pokemon 图片 / 头像、清晰的操作按钮。

现有页面能力包括：

- 速度线页：Pokemon / 形态选择、速度 SP、最终速度卡、速度散点图、benchmark 列表、收藏。
- 伤害计算页：进攻方 / 防守方卡片、从队伍选择、搜索图鉴、能力配置折叠区、性格 / 特性 / 道具 / 招式 / SP / 阶级、天气 / 场地 / 单双打、结果卡。
- 规则图鉴页：Pokemon / 招式 / 道具 / 特性 Tab、搜索、筛选、详情页。
- 队伍页：本地队伍、成员卡片、成员编辑底部 Sheet。

新原型不是重做这些工具页面，而是重排入口并补环境数据页面。进入具体工具后，应看起来像现有 PWA 的延续。

## 底部导航

底部导航调整为 4 个入口：

1. 环境
2. 队伍
3. 工具
4. 我的

默认进入“环境”。底部导航保留图标 + 文案 + 激活态的稳定移动端结构。

## 环境首页

环境首页是默认首页，服务浅度用户和高频回访用户。用户即使没有创建队伍，也应能打开应用获得价值。

首屏信息：

- 标题：环境
- 赛季与规则上下文，例如 `Season 1`
- 单打 / 双打胶囊切换，不能放成右上角“筛选”按钮
- 数据说明弱文本：`基于高分构筑样本统计 · 更新于 05-27 10:00`
- 数据来源入口作为弱文本链接，不要做成与单打 / 双打并列的胶囊

首页主模块：

- 热门宝可梦榜必须突出，是首页最高价值模块
- 展示排名、Pokemon 图片、名称、属性、样本占比、出现队伍数
- 宝可梦榜卡片内不展示道具
- 默认展示若干条，提供“查看全部”进入完整榜单

首页辅助模块：

- 热门队伍样本，展示 2-3 张高分队伍卡片
- 队伍卡片展示作者 / 分数 / 核心描述、6 只 Pokemon、两个按钮：`导入配置`、`队报链接`
- `队报链接` 直接打开外部原文，不进入中间页

不要在环境首页做：

- 我的队伍对照
- 全局道具榜
- 全局热门招式榜
- 强推荐式“热门核心 / 最佳搭档”大模块

## 完整宝可梦榜

点击环境首页“查看全部”进入完整宝可梦榜。完整榜单继续使用同一套单打 / 双打胶囊切换，不要改成两个独立按钮。

不要展示“Top 50”“点击条目查看详情”等解释性注释。展示数量和点击行为由产品默认规则决定。

点击榜单 Pokemon 进入新的“宝可梦环境详情页”，不要跳原有百科图鉴页。

## 宝可梦环境详情页

这个页面不是传统图鉴页，而是解释当前环境里这只 Pokemon 的表现。

模块顺序：

1. 顶部概览：Pokemon 图片、名称、属性、当前排名、样本占比、出现队伍数
2. 常用招式
3. 常见携带道具
4. 常同队出现
5. 相关高分队伍

交互规则：

- 常用招式 Top 10
- 常见携带道具 Top 10
- 常同队出现 Top 7
- 每个模块默认展示 Top 3，点击“查看全部”后在当前页面内展开，不要用底部抽屉，不要进入三级页面
- 点击招式或道具时，用条目内展开展示详情
- 点击常同队 Pokemon 可以进入该 Pokemon 的环境详情页，并保留返回路径

常用招式数据不是 Open Data 稳定字段，可以在原型中展示，但请不要把它画成官方完整数据。可以用“样本招式统计”一类中性表达。

## 高分队伍样本与导入

高分队伍卡片可出现在环境首页、宝可梦详情页和后续高分队伍浏览页。

卡片内容：

- 作者
- 分数 / 排名
- 队伍标题或核心描述
- 6 只 Pokemon 头像
- 按钮：`导入配置`、`队报链接`

导入配置行为：

- 点击 `导入配置` 后，把高分队伍复制为本地队伍
- 用户第一次点击 `导入配置` 时，先出现一次轻量提示：`目前可稳定带入 Pokémon 和道具；性格、SP、完整配招等信息可能缺失。需要原作者详细配置时，可以打开队报链接查看。`
- 首次提示只提供两个动作：`队报链接`、`继续导入`；用户继续导入后，后续导入不再重复弹出
- 跳转到“队伍”列表页
- 高亮新导入队伍 2-3 秒
- 不直接进入队伍详情或编辑页

如果字段缺失但能形成队伍骨架，仍允许导入，提示“已导入，可继续补全配置”。缺失项在队伍校验和成员编辑中处理。

只有完全无法识别 Pokemon、队伍为空、规则不匹配到无法转换时，才显示导入失败。

## 队伍页

队伍页默认显示队伍列表，不要默认进入某个队伍详情。

队伍来源包括用户手动创建、高分队伍导入、后续外部链接导入。不论来源如何，队伍列表卡片样式保持一致，只用标签或次要文案表达来源。

队伍卡片按钮统一为：

- 编辑配置
- 生成图片

生成图片是队伍分享功能：

- 点击后直接生成默认黑底 2x3 队伍图片
- 不做模板选择
- 不做字段选择
- 结果层只放图片和一个按钮：`保存图片`
- 图片内容包括队伍名称、6 只 Pokemon、Pokemon 图片、名称、属性、性格、特性、道具图标、道具名称、配招、生成时间戳

## 队伍详情 / 编辑

队伍详情定位为本地队伍工作台，不再强调来源。

不要在队伍详情页展示：

- 来源卡片
- 队报链接
- 生成图片 / 伤害计算 / 速度线快捷入口
- 顶部更多菜单

队伍标题可直接点击编辑。删除队伍放在页面底部危险区。

成员卡片沿用现有 PWA 队伍页样式：Pokemon 图片、名称、属性、道具图标、道具名称。成员卡片不直接展示招式，点开成员编辑后再展示招式。

成员编辑沿用当前 PWA 的底部 Sheet，不要改成新的“基础配置卡片网格”。结构：

- 顶部标题：编辑成员
- 关闭按钮
- 成员摘要卡：图片、名称、属性、道具
- 特性选择
- 道具搜索与列表
- 性格选择
- 4 个招式位
- SP 分配
- 校验结果
- 底部按钮：删除 / 取消 / 保存

添加成员：

- 队伍未满 6 人时显示空槽位 / 添加入口
- 点击空槽打开 Pokemon 选择 Sheet
- 选中 Pokemon 后直接进入成员编辑 Sheet

已有成员编辑时，可在摘要卡右侧提供轻量“更换”入口，回到 Pokemon 选择 Sheet 后继续同一个编辑流程。

## 工具页

工具页是三个并列入口，不要区分“常用 / 查询”，不要预设哪个更常用。

只展示三个入口：

- 伤害计算
- 速度线计算
- 规则图鉴

不要在工具首页展示：

- 当前规则入口
- 队伍配置带入说明模块
- 解释性注释

从队伍带入配置是具体工具内部能力。例如伤害计算页里可以“从队伍选择攻击方 / 防守方”，速度线计算页里可以“从队伍生成速度线”。临时修改不自动写回队伍。

## 我的页

“我的”是低频管理页，承接原设置能力，不抢环境、队伍、工具的注意力。

不展示本地队伍数、收藏数这类低价值概览。

页面结构：

- 偏好：主题、语言
- 本地资料：导出备份 JSON、导入备份
- 收藏入口
- 数据与缓存：数据来源、离线缓存、清除本地数据

导出语义必须区分：

- 队伍页的“生成图片”：用于分享展示，只针对单支队伍
- 我的页的“导出备份 JSON”：用于迁移 / 备份，应包含本地队伍、收藏、偏好设置、缓存索引等本地资料

不要把“当前规则”作为“我的”页强入口。规则、赛季、更新时间主要跟随环境页。

## 状态逻辑

环境页状态：

- 正常：展示榜单与热门队伍样本
- 加载中：展示骨架屏或轻量 loading
- 数据过期但可用：继续展示缓存内容，只在更新时间附近弱提示
- 离线但有缓存：继续展示内容，提示“离线模式 · 使用上次缓存”
- 离线且无缓存：展示空状态“无法加载环境数据”，提供“重试”
- 部分字段缺失：能展示的字段继续展示，不造假

高分队伍导入状态：

- 首次导入前：轻量提示当前只能稳定带入 Pokémon / 道具，并提供 `队报链接` 与 `继续导入`
- 成功：Toast “已导入配置”，跳转队伍列表并高亮新队伍
- 成功但字段缺失：Toast “已导入，可继续补全配置”，跳转队伍列表
- 失败：轻量错误层或 Toast，留在当前页面

## 视觉与交互要求

- 移动端优先
- 信息密度高但不拥挤
- 深色主题优先，可以保留现有 PWA 的卡片质感
- 不做营销 hero
- 不做大段说明文字
- 不把注释类文案放进界面
- 不要用卡片套卡片
- 不要使用过重装饰、渐变球、空洞氛围图
- 按钮文案必须是动作：导入配置、队报链接、编辑配置、生成图片、保存图片、导出备份 JSON、导入备份

请输出一套能展示主路径的移动端高保真原型，至少包含：

1. 环境首页
2. 完整宝可梦榜
3. 宝可梦环境详情页
4. 高分队伍卡片与导入状态
5. 队伍列表
6. 队伍详情 / 成员编辑 Sheet
7. 生成图片结果层
8. 工具页
9. 我的页
10. 离线 / 缓存 / 数据缺失状态示例
