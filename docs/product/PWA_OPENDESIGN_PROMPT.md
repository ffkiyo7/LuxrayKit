# OpenDesign Prompt：Luxray Kit 环境优先 PWA

更新日期：2026-06-11

## 使用说明

这是用于继续设计迭代的当前版提示词，不是实现规格的唯一来源。设计稿必须服从现有产品边界和代码状态。

## 产品

为 Luxray Kit 设计一套 390px 到 430px 宽的移动端优先 PWA。它是 Pokémon Champions 非官方对战助手，不是营销站，也不是完整战斗模拟器。

一级任务：

- 查看当前单打 / 双打环境样本。
- 查看宝可梦环境详情与公开上位构筑。
- 把队伍骨架导入本地并继续编辑。

二级任务：

- 查询规则图鉴。
- 进行实验性单次伤害计算。
- 管理主题、本地备份和离线缓存。

## 现有结构

- 底部导航：环境、队伍、工具、我的。
- 默认进入环境。
- 全局 Header 显示 Luxray Kit 和 Regulation Set M-A。
- 深色主题以蓝黑、黄色电光和高对比卡片为主，同时支持浅色主题。
- 内容最大宽度 430px，需处理安全区。

## 必须设计的页面

1. 环境首页：单双打切换、数据来源、更新时间、Top 4 宝可梦、上位构筑、数据口径入口。
2. 完整宝可梦榜。
3. 宝可梦环境详情：常用招式、携带道具、常见队友、相关构筑。
4. 数据口径页：样本池、来源、分母说明、导入字段限制。
5. 队伍列表与队伍详情。
6. 成员编辑弹层和 SP 调整弹层。
7. 工具页：规则图鉴、伤害计算、禁用态速度线卡。
8. 规则图鉴列表、筛选和详情。
9. 伤害计算的攻防选择、配置和结果。
10. 我的：主题、备份、离线缓存、清除本地数据。
11. 首次环境导入提示、成功 Toast、加载失败和离线回退状态。

## 文案边界

- 使用“样本占比”“上位构筑快照”，不要使用“官方使用率”。
- 导入提示必须说明只稳定带入宝可梦和道具。
- 伤害结果必须保留“实验性主线近似”。
- 速度线卡显示“敬请期待”和“未开放”，不可设计成可点击页面。
- 不把静态回退、Worker 缓存或社区资料包装成官方实时数据。

## 交互要求

- 排名、详情和队伍卡保持高信息密度，但触控目标不得过小。
- 长列表滚动时允许底部导航自动隐藏。
- 模块展开在页面内完成，不增加无必要的三级页面。
- 弹层打开时阻止底部导航误触。
- 所有在线数据页面都要有静态回退、过期缓存和失败状态的视觉方案。

## 不要设计

- 账号、登录、云同步和社交社区。
- 完整战斗回合模拟。
- 可操作的速度线入口。
- 将公开队报聚合数据自动填充为某支队伍的原作者完整配置。
