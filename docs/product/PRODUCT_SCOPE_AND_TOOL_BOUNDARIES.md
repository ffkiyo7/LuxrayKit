# 产品范围与工具边界

更新日期：2026-06-11

## 当前主线

Luxray Kit 优先解决：

1. 用公开上位构筑样本快速理解当前环境。
2. 查看并导入公开队报中的队伍骨架。
3. 在设备本地维护可编辑、可备份、可分享的队伍。

环境页、环境详情、上位构筑和队伍编辑是一级能力。

## 二级工具

### 规则图鉴

当前已开放。它用于查询本地 Regulation Set M-A catalog，不代表所有条目都已获得官方逐项确认。

### 伤害计算

当前已开放，但结果必须标记为实验性主线近似。

允许：

- 单次伤害招式计算。
- 从队伍或图鉴选择攻防双方。
- 显式设置形态、特性、道具、性格、SP、能力阶级、天气、场地、HP、异常状态、会心、双打分摊和保护。
- 处理已在 adapter 中建模的直接伤害特性、道具与招式规则。

不允许包装成已支持：

- 完整回合、换人、先后手、命中率、随机触发或队友行动。
- 自动触发威吓、天气手、速度变化或多回合状态。
- 对未确认 Champions 机制给出官方结论。

### 速度线

Champions SP 速度计算代码、测试和旧页面仍存在，但 `ToolsPage` 的入口已禁用。产品文档、截图和测试都应把它描述为“未开放”，不能写成当前可用功能。

## 环境数据边界

- 在线优先读取同源 Worker API。
- Worker 缓存 PokeDB ranked-team Open Data 和可解析到的队报样本。
- 静态 S1 快照负责离线和 API 故障回退，并额外包含本地维护脚本生成的招式统计。
- UI 中的“样本占比”以当前样本池为分母，不使用“官方使用率”措辞。
- 从环境导入时只承诺宝可梦和道具；其他字段需要用户确认。

## 新需求判断

新功能只有在满足以下条件时才进入主线：

- 明显提升环境理解、构筑参考或本地队伍维护。
- 数据来源和缺失字段可以公开说明。
- 离线回退不会被破坏。
- 不要求把伤害计算扩展成完整战斗模拟器。

账号、云同步、实时全服统计、多赛季趋势和完整战斗状态机均不在当前范围。
