# PWA 环境优先重构需求与实现对照

更新日期：2026-06-11

## 文档状态

本文件最初是环境优先重构的需求输入。重构已合入 `main`，以下内容以当前代码实现为准；旧草图不再作为未完成任务清单。

## 已实现的信息架构

- 430px 移动端外壳。
- 底部 4 Tab：环境、队伍、工具、我的。
- 默认入口为环境。
- 页面使用 React lazy loading；环境数据、队伍导入 helper 和计算引擎按需加载。
- 环境与图鉴长页面滚动时，底部导航可自动隐藏；弹层打开时锁定。

## 环境页

### 首页

- 单打 / 双打切换。
- 来源和更新时间。
- 宝可梦榜 Top 4 与完整榜单。
- 上位构筑每批 4 条，可“换一批”。
- “数据口径”独立页面。

### 环境详情

- 宝可梦概览、样本占比和队伍数。
- 常用招式默认 5 条、最多 10 条。
- 携带道具默认 5 条、最多 10 条。
- 常见队友默认 4 条、最多 7 条。
- 相关上位构筑、队报链接和导入按钮。

### 数据加载

1. 请求 `/api/environment/latest`，使用 `cache: no-store`。
2. 请求失败时读取 `/data/pokedb/reg-ma-s1-environment.json`，使用 `force-cache`。
3. 两者都失败时回退到 `environmentDatasetSeed`。

Worker 使用 KV 保存最新 snapshot、状态和宝可梦队伍索引，Cron 为 `17 */6 * * *`。它按 `SEASON_CANDIDATES=2,1` 依次尝试赛季，只有单打和双打都可用时才采用该赛季。

## 队伍页

- 列表优先，支持创建、编辑、删除和排序。
- 成员编辑使用移动端弹层，包含形态、特性、道具、性格、招式和 SP。
- 环境导入会创建本地副本，跳转队伍页并短暂高亮。
- 第一次导入前显示字段覆盖提示；确认后写入偏好，不重复打断。
- 分享图为 2x3 队伍卡，并使用本地时区生成时间戳。

## 工具页

- 规则图鉴：已开放。
- 伤害计算：已开放。
- 速度线计算：入口已禁用，显示“敬请期待 / 未开放”。
- 当前规则不是工具卡；规则摘要由 Header 入口打开。

## 我的

- 深色 / 浅色主题。
- 完整 JSON 备份导出与导入。
- 兼容旧版仅队伍 JSON。
- 离线缓存说明。
- 清除本地数据。

## 与早期原型不同的地方

- 速度线不再是当前开放入口。
- 环境在线数据已接入 Worker，不再只是静态 JSON。
- Worker 已部署到生产域名并由 GitHub Actions 自动发布。
- 视觉回归当前为 13 个截图状态，不包含速度线页面。
- 环境加载失败时当前 AppShell 显示失败提示；没有单独“重试”按钮。
- Worker snapshot 目前不包含静态维护包的完整 `moveStats`，所以在线数据和离线静态包的详情丰富度可能不同。
