# 机制边界

更新日期：2026-06-11

## 已用于产品

- Regulation Set M-A 基础规则、Mega 开关和队伍限制。
- 当前 catalog 中的宝可梦、形态、属性、种族值、招式、道具、特性与 learnset。
- Champions Lv.50 SP v1 能力值计算。
- 本地队伍的基础合法性和数据完整性检查。
- 单次伤害的 Gen9 主线近似。

## 速度

`src/lib/calculations.ts` 已实现并测试 Champions SP 速度计算、性格和顺风修正。旧 `SpeedPage` 和 benchmark 类型仍保留。

产品状态不同于底层状态：当前工具页速度线入口已禁用，因此不能把它描述为已交付给用户的功能。

## 伤害

当前 adapter 使用：

- `@smogon/calc` Gen9 作为主线公式基础。
- 项目内 Champions 招式参数、形态和能力值。
- 用户显式输入的天气、场地、能力阶级、HP、异常状态、会心、保护和双打分摊。
- 已通过测试的直接伤害特性、道具和招式效果。

部分 Champions 新增 Mega 特性已作为直接伤害行为覆盖；需要队友、换人、命中率、受击后状态、濒死或多回合上下文的特性只记录边界，不自动模拟。

## 未确认或不建模

- Champions 是否在所有取整、随机数和边界条件上完全等同 Gen9 主线。
- 完整回合、换人、速度顺序和队友行动。
- 概率触发、命中、重定向、替身和墙的完整状态。
- 一场战斗内特性触发次数和跨回合持续效果。
- 未经数据源确认的合法性、招式或专属机制。

## 处理原则

- 已验证的直接效果才进入 adapter。
- 需要战斗流程的效果不因“可以推测”而自动启用。
- 不支持的输入必须显式阻断或提示。
- 机制研究与产品入口分开：底层代码存在不等于用户功能已开放。
- 所有强结论必须能追溯到来源或回归测试。
